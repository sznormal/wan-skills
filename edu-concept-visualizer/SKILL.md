# 教学概念可视化器 (Edu Concept Visualizer)

## 简介

**教学概念可视化器** 是一个专为教育工作者设计的智能视觉创作 Skill，能够将抽象的教学概念一键转化为生动的可视化素材（图片/视频），支持多种教学场景，让课堂讲解更直观、更吸引学生。

## 核心价值

### 解决的痛点
- **抽象概念难理解**：数学公式、物理原理、化学反应等抽象概念学生难以想象
- **备课素材耗时长**：教师需要花大量时间搜索或制作教学图示
- **视觉素材质量低**：网上找的图片风格不统一、版权不清晰
- **动态演示门槛高**：需要专业软件才能制作动态讲解视频

### 创新亮点
1. **比喻式可视化**：用生活化场景比喻抽象概念（如"电流=水流"）
2. **分步动态演示**：自动生成分步骤讲解的视频
3. **教学场景优化**：针对课堂演示场景设计输出格式
4. **多学科覆盖**：支持数理化生、历史地理等多学科概念

## 能力矩阵

| 能力 | 输入 | 输出 | 适用场景 |
|------|------|------|----------|
| 概念比喻图 | 抽象概念描述 | 生活化比喻图片 | 导入新课、概念引入 |
| 原理分解图 | 复杂原理描述 | 分步骤静态图解 | 知识讲解、板书 |
| 动态演示视频 | 流程/变化描述 | 15-30秒动态视频 | 实验演示、过程讲解 |
| 故事板讲解 | 知识点+故事线 | 多格故事板 | 情境教学、案例分析 |
| 概念对比图 | 两个概念对比 | 对比式视觉图 | 易混淆概念区分 |

## 触发条件

### 正向触发（应该调用）
- 用户描述一个抽象概念需要可视化
- 用户提及"帮我把...做成图/视频"
- 用户说"学生理解不了...怎么讲更直观"
- 用户提到"备课"、"教学素材"、"课堂演示"
- 用户描述需要对比两个概念

### 负向触发（不应调用）
- 用户只是询问概念定义（不需要可视化）
- 用户要求制作PPT（应使用ppt相关Skill）
- 用户要求生成习题（应使用习题相关Skill）

## 执行步骤

### Step 1: 意图识别
解析用户输入，识别：
- 概念类型（定义类/原理类/过程类/对比类）
- 学科领域（数学/物理/化学/生物/历史/地理等）
- 输出偏好（静态图/动态视频/故事板）
- 目标受众（小学/初中/高中/大学）

### Step 2: 提示词构建
根据概念类型选择合适的提示词模板：
- **定义类**：使用比喻可视化模板
- **原理类**：使用分解图模板
- **过程类**：使用动态演示模板
- **对比类**：使用对比图模板

### Step 3: API调用
调用阿里万相Wan 2.7 API：
- 静态图：调用 `wan2.7-image-pro` 模型
- 动态视频：调用 `wan2.7-t2v` 模型
- 带文字渲染：利用3K Token文字能力

### Step 4: 结果处理
- 保存生成的图片/视频到指定目录
- 返回文件路径和预览链接
- 提供使用建议（适合的教学场景）

### Step 5: 迭代优化
- 根据用户反馈调整提示词
- 支持重新生成或局部修改
- 记录成功的提示词模板供复用

## 配置说明

### 必需配置
```bash
export WAN_API_KEY="your-api-key"
export WAN_BASE_URL="https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation"
```

### 可选配置
```bash
export EDU_OUTPUT_DIR="./edu_visuals"  # 输出目录
export EDU_DEFAULT_STYLE="illustration"  # 默认风格
```

## 使用示例

### 示例1：概念比喻可视化
```
用户：帮我把"电流"这个概念做成图，让学生更好理解
执行：
1. 识别为"定义类"概念，物理学科
2. 选择比喻模板：电流=水流管道系统
3. 调用wan2.7-image-pro生成
4. 返回：一张展示"水流管道系统"与"电路系统"对比的示意图
```

### 示例2：原理分解图
```
用户：光合作用的过程怎么画更清晰？分步骤展示
执行：
1. 识别为"原理类"，生物学科
2. 选择分解图模板：光反应+暗反应
3. 调用wan2.7-image-pro生成组图
4. 返回：3张分步骤讲解图（光反应阶段、暗反应阶段、整体循环）
```

### 示例3：动态演示视频
```
用户：帮我做一个"水循环"的动态讲解视频，30秒左右
执行：
1. 识别为"过程类"，地理学科
2. 构建动态演示提示词
3. 调用wan2.7-t2v生成视频
4. 返回：展示蒸发→凝结→降水的动态视频
```

### 示例4：故事板讲解
```
用户：用故事的方式讲"勾股定理"，做成故事板
执行：
1. 识别为"故事板"需求，数学学科
2. 构建故事线：古埃及测量→发现规律→证明定理
3. 调用wan2.7-image-pro生成组图
4. 返回：4格故事板（发现场景、推导过程、证明方法、应用示例）
```

## 失败处理

### API调用失败
- 检查API Key是否正确配置
- 检查网络连接
- 重试最多3次，间隔递增

### 生成效果不满意
- 提示用户补充更多描述
- 提供风格选择（卡通/写实/科学风格）
- 支持局部修改指令

### 不支持的概念类型
- 说明当前能力边界
- 建议用户换一种描述方式
- 记录需求供后续迭代
